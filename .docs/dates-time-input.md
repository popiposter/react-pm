# Time inputs

## Текущее решение

В проекте больше не используется Mantine `TimeInput`.

Для ввода времени в редакторе табеля применяются нативные поля `input[type="time"]`. Это намеренное решение:

- меньше зависимостей
- лучше совместимость с mobile браузерами
- естественное поведение для PWA
- проще интеграция в локальный UI-kit

## Почему этого достаточно

Сценарий приложения не требует сложного time picker с кастомным dropdown. Пользователю нужно быстро вводить начало и конец рабочего интервала, а нативный control решает эту задачу без лишнего веса.

## Ограничения

- формат отображения зависит от браузера и locale
- браузер сам контролирует вид встроенного picker UI
- секунды сейчас не используются

## Если позже понадобится улучшение

Тогда стоит рассмотреть:

- отдельный masked input
- кастомный picker только для desktop
- сохранение mobile native UX

Но до появления реального product requirement усложнять time input не нужно.
